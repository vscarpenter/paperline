# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

- `npm run build` — runs `scripts/build.mjs`, which regenerates the three browser-global files in `dist/` from `src/`. **Always run this after editing `src/components.jsx`, `src/icons.jsx`, or `src/styles/paperline.css`.** Never edit anything inside `dist/` by hand — it is overwritten.
- `npm test` — runs `scripts/check-package.mjs`. Verifies required files exist, that `docs/index.html` still references the three `../dist/paperline*` artifacts, that `dist/` matches a fresh build of `src/` (`scripts/build.mjs --check`), that every `PL*` exported from `src/components.jsx` is also re-exported by `src/index.jsx` and assigned to `window` in `dist/paperline-components.global.jsx`, and that every icon key in `src/icons.jsx` is present in the dist icons file. It is a structural + parity check, not a unit-test runner.
- `npm run docs` — serves the repo root via `python3 -m http.server 8000`. Open `http://localhost:8000/docs/index.html` for the full spec or `http://localhost:8000/examples/browser.html` for the minimal integration.
- There is no lint and no test framework. Source files are shipped as JSX.

## Architecture

Paperline is a static-first design-system package with a **dual distribution** that must be kept in sync manually:

- `dist/esm/` and `dist/cjs/` are the compiled outputs consumed by bundled React apps (`exports` in `package.json` points here). `src/index.jsx` re-exports `Icon`, `I`, and the `PL*` primitives and is compiled by esbuild during `npm run build`. React is a peer dependency — never add it as a runtime dep. No bundler configuration is required by consumers.
- `dist/` contains browser-global twins of the same code, **generated by `scripts/build.mjs` from `src/`** — do not edit `dist/*` by hand. The build strips `import`/`export`, replaces `import { useState, ... } from "react"` with `const { useState, ... } = React`, and converts the trailing `if (typeof window…)` guard into a flat `Object.assign(window, …)`.
- `dist/paperline.css` is a generated copy of `src/styles/paperline.css`. `npm test` re-runs the build in `--check` mode and fails on any drift, so the previous "remember to mirror by hand" footgun is gone.

### Tokens and theming

All visual decisions flow through CSS custom properties defined in `src/styles/paperline.css`:

- Surfaces (`--pl-paper-*`), ink (`--pl-ink-*`), accent plum (`--pl-accent-*`), muted status colors, categorical chart palette, type stack, 4px-rhythm spacing, radius, shadows, motion, z-index, semantic form tokens (`--pl-field-*`), and a numeric type scale (`--pl-text-*`).
- Components apply tokens through inline `style={{ background: "var(--pl-accent-500)", … }}` — they have no className-based theming for color. Never hardcode hex colors in components; always reference `var(--pl-*)`.
- Components do, however, attach a small set of utility classNames (`pl-btn`, `pl-field`, `pl-tab`, `pl-iconbtn`, `pl-menu-item`, `pl-link`, `pl-page-btn`, `pl-checkbox`, `pl-radio`, `pl-toggle`) that the stylesheet uses to wire `:focus-visible` to `--pl-focus`. Pseudo-classes can't live in inline styles, so this hybrid is intentional — keep these classNames in sync when adding new interactive primitives.
- The stylesheet honours `prefers-reduced-motion` and exposes a `.pl-sr-only` visually-hidden utility used by the form primitives.
- Dark mode is achieved by adding `pl-dark` (or `data-theme="dark"`) to a wrapping element; the CSS file rebinds the same variables under that selector. New tokens must define both light and dark values.

### Icons

`src/icons.jsx` exports a single `Icon` wrapper (24x24 viewBox, 1.6 stroke, `currentColor`) and an `I` namespace object whose keys return small JSX helpers like `I.Plus`, `I.Search`. Add new icons as members of `I` so consumers can pass them as props (`<PLButton icon={I.Plus} />`). Mirror additions into `dist/paperline-icons.global.jsx`.

### Components

`src/components.jsx` exports `PL`-prefixed primitives. Current set: `PLButton`, `PLInput`, `PLTextarea`, `PLSelect`, `PLToggle`, `PLCheckbox`, `PLRadio`, `PLField`, `PLBadge`, `PLTag`, `PLCard`, `PLDivider`, `PLAvatar`, `PLAvatarGroup`, `PLProgress`, `PLSpinner`, `PLSkeleton`, `PLAlert`, `PLEmptyState`, `PLTabs`, `PLStat`, `PLTable`, `PLToast`, `PLStack`, `PLCluster`, `PLGrid`, `PLBreadcrumbs`, `PLPagination`, `PLTooltip`, `PLMenu`, `PLModal`. Conventions in this file:

- Variant maps live as plain objects at the top of each component (`const kinds = { primary: {...}, ghost: {...} }`); add new variants by extending the map, not by branching.
- Sizing maps follow the same pattern (`sm`/`md`/`lg`).
- Form primitives (`PLToggle`, `PLCheckbox`, `PLRadio`) wrap a real `<input>` hidden with `.pl-sr-only` so they remain keyboard-operable and screen-reader-announced. Don't replace them with bare `<span>`s.
- Overlays (`PLModal`) lock `document.body.style.overflow`, restore focus to the previously focused element on close, and run a small Tab-cycle focus trap. `PLMenu` registers document-level click + Escape handlers while open and clears them on unmount.
- Add new component exports to (a) the `export { … }` block, (b) the `Object.assign(window, …)` block, and (c) `src/index.jsx`. The check script verifies all three are in sync.
- After changing behavior here, run `npm run build && npm test` — never edit `dist/*` by hand.

## Conventions

- Two-space indentation across CSS and JSX.
- CSS tokens are prefixed `--pl-` and grouped by purpose with banner comments.
- React primitives use the `PL` prefix; icons live under the `I` namespace.
- Plum is the only primary accent and should be used sparingly; status colors are muted earth tones, never saturated.
- For component changes, add or update a visible example in `docs/index.html` so reviewers can inspect default, hover/focus, disabled, and error states.

## Package surface

`package.json` `exports` map (do not break without a version bump):

- `.` → `./src/index.jsx`
- `./styles.css` → `./src/styles/paperline.css`
- `./components`, `./icons` → matching `src/*.jsx`
- `./dist/paperline.css`, `./dist/components.global.jsx`, `./dist/icons.global.jsx` → the browser-global artifacts
